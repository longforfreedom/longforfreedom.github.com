Spark学习-Windows下环境搭建

By migle

11月 5 2014 更新日期:11月 7 2014

文章目录
  1. 1. 环境
  2. 2. 构建Spark
    1. 2.1. 导入源码

本说描述了spark在Win 8下通过IntelliJ IDEA从github下载代码、构建、运行、测试的详细过程,此过程涉及的相关软件的安装及配置不再详细描述

环境

  1. 操作系统：Win8 64
  2. JDK：1.8
  3. maven:3.0.2

    配置好M2_HOME后，IDEA会自动检测到maven

  4. git:1.9.0
  5. IntelliJ IDEA 13.1.5

构建Spark

导入源码

注意:这里需要安装好git，也可以直接下载Spark源码包，导入到IDEA中

  • 在IDEA中”VCS”->”checkout from version control” -> “github” 输入Spark在github上的仓库地址 https://github.com/apache/spark.git如下图：

  • 等待Spark源码下载完成会提示是否要打开”pom.xml” 点”yes” 然后等待Spark工程导入完毕，打开“project”（快捷键：alt+1）如下图所示:

构建

  1. 命令行中构建
    mvn -Phadoop-2.4 -Dhadoop.version=2.4.0 -Phive -DskipTests clean package

  2. IDEA中构建

    • maven设置 打开”file” -> “setting” -> “maven” -> “runner” 选中”skip tests”
    • 选择Profiles 打开”view” -> “tool windows” ->”maven projects” 根据需要选择Profiles：
    • 构建 在”maven projects”打开”spark project parent POM” ->”lifecycle” 选择“run maven build” 等待执行结果
    • 构建结果

构建成功时会在目录assembly/target/scala-2.10下生成Jar包

可以切到某一个稳定的分支上比如1.1上
IDEA中: 点击右下角状态栏上的”git:master” 选择特定分支后选择”checkout as new local branch”
命令行中： git checkout -b branch-1.2 remotes/origin/branch-1.2

Spark包括多个模块，完整编译一次时间较长，可以修改pom.xml将其<modules>元素中不观注的模块注释掉，然后构建
IDEA中方法不变，命令行中可以用mvn -Dhadoop.version=2.4.0 -DskipTests clean package 可以裁剪到只保留core和其依赖即可

  • 测试程序

写一个简单的测试程序,在Spark-core模块中创建FirstSpark,代码如下:

import org.apache.spark.{SparkContext, SparkConf}

/**
 * Created by migle on 2014/11/7.
 */
object FirstSpark {
  def main(args: Array[String]) {
    val conf = new SparkConf().setMaster("spark://migle-ailk:7077").setAppName("FirstApp");
    val sc = new SparkContext(conf)
    val rdd  = sc.textFile("e:/spark/spark/README.md").cache()
    val lines = rdd.count()
    println("lines:%s".format(lines));
  }
