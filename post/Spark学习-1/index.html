环境

下载： wget xxxxxxxxx

单机：

    $SPARK_HOME/sbin/start-master.sh
    $SPARK_HOME/bin/spark-class org.apache.spark.deploy.worker.Worker spark://192.168.242.130:7077

集群：

  1. 集群各机器SSH免密已配置 参考
  2. 修改 $SPARK_HOME/conf/slaves 添加所有slave，每行一条
  3. 修改 $SPARK_HOME/conf/spark-env.sh 添加：SPARK_MASTER_IP=MASTER的IP
  4. scp到其它机器
  5. 在Master上启动集群 $SPARK_HOME/sbin/start-all.sh

浏览器中查看：http://192.168.242.130:8080/
默认端口7077可以在$SPARK_HOME/spark-env.sh通过设置SPARK_MASTER_PORT修改

sbt安装：

  1. 下载：

     cd /usr/bin  
 wget http://repo.typesafe.com/typesafe/ivy-releases/org.scala-sbt/sbt-launch/0.13.2/sbt-launch.jar

  2. vim /usr/bin/sbt 写入以下内容

     SBT_OPTS="-Xms512M -Xmx1536M -Xss1M -XX:+CMSClassUnloadingEnabled -XX:MaxPermSize=256M"
 java $SBT_OPTS -jar `dirname $0`/sbt-launch.jar "$@"

  3. chmod +x /usr/bin/sbt

spark-submit

可以不用先启动Master或集群环境
关键代码 可以不用设置Master URL

val conf = new SparkConf().setAppName("Simple Application")
val sc = new SparkContext(conf)
val rdd = sc.textFile(args(0)) //.cache()
val lines  = rdd.count();
println("总共：" + lines)

运行：
打包：sbt package

sbt中运行

首先需要启动Master和Worker

关键代码：必须要设置Master URL

    val conf = new SparkConf()
                            .setAppName("Simple Application")  
                            .setMaster("spark://192.168.242.130:7077") 
                            .set("spark.executor.memory","100m") //"spark.executor.memory"值不能大于SPARK_WORKER_MEMORY中的值,根据实际情况设置
                            //set("spark.driver.host", "192.168.242.1")
    val sc = new SparkContext(conf)

运行：sbt run

注意：

  1. 可能需要设置一下 $SPARK_HOME/spark-env.sh 在其中添加以下
    SPARK_WORKER_MEMORY=1000m ## 根据实际情况
    SPARK_MASTER_IP=192.168.242.130 ## 最好直接设置成Master的IP，否则会取成hostname
  2. SPARK_WORKER_MEMORY的值不能小于SparkConf中spark.executor.memory中设置的值，否则会出现后面的错误，spark.executor.memory默认会取512MB，代码如下:
//SparkContext.scala
private[spark] val executorMemory = conf.getOption("spark.executor.memory")
.orElse(Option(System.getenv("SPARK_EXECUTOR_MEMORY")))
.orElse(Option(System.getenv("SPARK_MEM")).map(warnSparkMem))
.map(Utils.memoryStringToMb)
.getOrElse(512)
  1. Driver如果和Master没在同一机器上时可能需要修改一下hosts文件，因为Driver程序取的主机名格式为 xxxx.mshome.net 而不是IP，所以需要在Master的hosts文件添加Driver和主机名和IP的映射，即使在设置了spark.driver.host也不启作用,原因后缀分析

否则会出现以下错误：

14/08/28 14:05:42 WARN TaskSchedulerImpl: Initial job has not accepted any resources; check your cluster UI to ensure that     workers are registered and have sufficient memory
14/08/28 14:05:55 ERROR SparkDeploySchedulerBackend: Application has been killed. Reason: All masters are unresponsive! Giving     up.  
14/08/28 14:05:55 INFO TaskSchedulerImpl: Removed TaskSet 0.0, whose tasks have all completed, from pool  
14/08/28 14:05:55 INFO TaskSchedulerImpl: Cancelling stage 0  
14/08/28 14:05:55 INFO DAGScheduler: Failed to run count at ETL.scala:22
[error] (run-main-0) org.apache.spark.SparkException: Job aborted due to stage failure: All masters are unresponsive! Giving up.

或一直循环打印类似下面信息

14/10/30 12:02:25 WARN scheduler.TaskSchedulerImpl: Initial job has not accepted any resources; check your cluster UI to ensure that workers are registered and have sufficient memory
