本文是CentOS6.5下python2.7,numpy,scikit-learn、ipython-notebook安装配置记录，CentOS6.5自带的是python2.6也可以,如果你不想升级到python2.7,可以试着直接安装打好包的scikit-learn。

Python2.7 安装

安装依赖

yum groupinstall "Development Tools"
yum install blas-devel lapack-devel atlas atlas-devel readline-devel sqlite-devel zlib-devel openssl openssl-devel

安装Python2.7

  • 下载：wget https://www.python.org/ftp/python/2.7.6/Python-2.7.6.tgz

  • 安装: 

      tar xjvf Python-2.7.6.tgz 
  cd Python-2.7.6.tgz     
  ./configure --enable-shared --prefix=/usr/local/python27 --enable-shared  
  make && make install

  • 配置：

      echo '/usr/local/python27/lib/' >> /etc/ld.so.conf
  /sbin/ldconfig             
  /sbin/ldconfig -v
  rm /usr/bin/python    
  #把/usr/local/python27/bin/加入PATH中也可以,
  ln -s  /usr/local/python27/bin/python2.7   /usr/bin/python
  ln -s /usr/local/python27/bin/python2.7    /usr/bin/python2.7 
  ln -s /usr/local/python27/bin/python2.7    /usr/bin/python2

pip安装:

    wget https://bootstrap.pypa.io/get-pip.py -O - | sudo python 
    #加环境变量，不喜欢这样也可以做链接
    echo 'export PYTHON_HOME=/usr/local/python27' >> /etc/profile
    echo 'export PATH=$PATH:$PYTHON_HOME/bin' >> /etc/profile
    #立即生效
    . /etc/profile

其它

  • 把/usr/bin/yum的第一行修改成#!/usr/bin/python2.6否则yum不能用

cikit-learn 安装

安装依赖

yum install blas-devel lapack-devel atlas atlas-devel readline-devel sqlite-devel  
yum install freetype-devel libpng-devel

不明白不为什么不复制到/usr/local/lib下面安装numpy的时候编译ld老是报错不到链接不到-lpython2.7
cp /usr/local/python27/lib/libpython2.7.so.1.0 /usr/local/lib
cd /usr/local/lib
ln -s libpython2.7.so.1.0 libpython2.7.so
检查：
whereis libpython2.7.so.1.0
gcc -lpython2.7 --verbose

安装scikit-learn

    pip install numpy
    pip install scipy
    pip install matplotlib
    pip install -U scikit-learn
    pip install ipython[notebook]  ##这个很方便，推荐安装

远程访问iPython

  • 生成密码：
    运行ipython输入以下：

会有输入密码提示，输入密码后会有类似‘sha1:cdadcc8c28ad:918a0295e7efc42466ce9c51a53b993650ae3776’信息输出，复制后面有用

* 生成密钥：
  • 创建配置文件：
    ipython profile create nbserver 看输出信息，会创建几个文件
    在~/.ipython/profile_nbserver/ipython_config.py添加以下
        # Kernel config
        c.IPKernelApp.pylab = 'inline'  # if you want plotting support always
        # Notebook config
        c.NotebookApp.certfile = u'/root/.ipython/mycert.pem'    #######前面openssl生成的文件绝对路径 
        c.NotebookApp.ip = '*'
        c.NotebookApp.open_browser = False
        c.NotebookApp.password = u'sha1:cdadcc8c28ad:918a0295e7efc42466ce9c51a53b993650ae3776'  #前面passwd()生成的密码串
        # It is a good idea to put it on a known, fixed port
        c.NotebookApp.port = 9999
  • 启动 ipython notebook --profile=nbserver
  • 远程测试
    https://192.168.242.129:9999

    本文中的CentOS是在虚拟机中安装的，192.168.242.129是虚拟机IP，在Win8测试
    会有证书不安全的提示，忽略之，要输入密码就是前面passwd()的时候输入的密码
